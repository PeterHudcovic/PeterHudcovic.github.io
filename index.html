DevOps • Linux • Cloudflare • Infrastructure Security • NGINX • Docker • Kubernetes • Apache

This is my technical sandbox where I test server configurations, security settings, deployment workflows, and performance optimizations. What started as a Linux, Apache, and Cloudflare setup has grown into a broader hands-on lab that now includes Kubernetes on DigitalOcean, Helm-based deployments, NGINX Ingress, secure API publishing, a Python FastAPI service, and a Java Spring Boot API connected to PostgreSQL through JDBC. I also moved website versioning and deployment into GitHub, so changes are now tracked, versioned, and deployed automatically instead of being edited manually on the server.

Cloudflare + Origin Lockdown

Full (Strict) TLS, origin certificates, DNS protection, and secure traffic flow through Cloudflare-managed ingress paths. This environment gives me a practical lab for certificate validation, protected origin exposure, WAF-related controls, and secure service publishing.

Kubernetes + Secure API Delivery

I deployed workloads on DigitalOcean Kubernetes, exposed services through NGINX Ingress, connected custom subdomains through Cloudflare, and published secure HTTPS endpoints with strict TLS validation.

Python FastAPI Service

Built and deployed a FastAPI application behind a custom domain with health checks, API endpoints, containerized delivery, and Helm-based rollout management. This gives me a working sandbox for ingress, DNS, TLS, and deployment troubleshooting.

Java JDBC + PostgreSQL Lab

Built and deployed a Spring Boot application connected to PostgreSQL through JDBC for database integration, backend connectivity testing, SQL execution, and troubleshooting across both the application and data layers.

Apache HTTP/2 + MPM Event

Optimized Apache with HTTP/2, HTTPS enforcement, and hardened web server settings. This remains part of the core Linux hosting layer behind the broader lab environment and helped form the base for later automation and deployment improvements.

GitHub Versioning + Automated Deployments

The website is now versioned through GitHub, with changes tracked through commit history and deployed automatically through GitHub Actions. This replaced manual server-side editing and gives me a cleaner, repeatable deployment flow for future site updates.

SSH Hardening

Key-based authentication, disabled root login, controlled firewall rules, and hardened SSH access for safer remote administration. This is one of the security foundations of the entire sandbox.
