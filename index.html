DevOps • Linux • Cloudflare • Infrastructure Security • NGINX • Docker • Kubernetes • Apache

This is my technical sandbox where I test server configurations, security settings, deployment workflows, failover behavior, and performance optimizations. What started as a Linux, Apache, and Cloudflare setup has expanded into a broader hands-on environment that now includes a three-node Kubernetes cluster on DigitalOcean, secure API publishing through NGINX Ingress, a Python FastAPI service, a Java Spring Boot API connected to PostgreSQL through JDBC, and persistent PostgreSQL storage backed by DigitalOcean Block Storage. Website changes are now versioned in GitHub and deployed automatically through GitHub Actions instead of being edited manually on the server.

Current Architecture

Main website traffic is served by Apache on a Linux server, while API traffic is routed through Cloudflare into a DigitalOcean Kubernetes cluster. The Java API connects to PostgreSQL through JDBC, and the PostgreSQL layer now uses persistent block storage.

Cloudflare + Origin Lockdown

Full (Strict) TLS, origin certificates, Cloudflare-managed DNS, and protected origin exposure. This setup is used to validate certificate behavior, TLS routing, ingress access, and secure service publishing across the website and Kubernetes-hosted APIs.

Kubernetes + Secure API Delivery

Containerized services deployed on DigitalOcean Kubernetes, exposed through NGINX Ingress and secured with Cloudflare-managed subdomains and TLS. This gives me a practical lab for ingress routing, service exposure, hostname-based traffic handling, and failover testing.

Python FastAPI + Java JDBC Lab

Built and deployed a FastAPI service for secure API delivery and a Spring Boot service connected to PostgreSQL through JDBC for database integration, query execution, backend troubleshooting, and real deployment testing across both the application and data layers.

Persistent PostgreSQL Storage

Migrated PostgreSQL from a temporary pod-based setup to persistent storage backed by DigitalOcean Block Storage. The database now runs as a StatefulSet with a PersistentVolumeClaim, restored data, and storage that survives pod recreation and restart events.

Apache HTTP/2 + GitHub Versioning

Apache remains part of the Linux hosting layer and was used for HTTPS enforcement, HTTP/2, and web server hardening. The website itself is now versioned through GitHub, with updates tracked through commit history and deployed automatically through GitHub Actions.

Kubernetes Failover Testing

Tested both controlled node draining and a full node power-off scenario. Kubernetes successfully rescheduled workloads onto other nodes, updated endpoints, restored ingress, and recovered API access after temporary interruptions. This lab now includes practical failover validation rather than only static deployment.

SSH Hardening

Key-based authentication, controlled firewall rules, and hardened remote access for safer server administration and deployment operations across the Linux hosting layer and the wider lab environment.
